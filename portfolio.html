MMS REST Microservices

Development of REST microservices for a mini-grid management. Having served as utility manager at Rubitec Gbamugbamu Microservices, I realised some challenges and took a confident step of writting server side solution for the problem as REST microservices. The project named "MMS" is developed to solve challenges of persisting data in to cloud. Resources such as shutdown, expenditure, generator runtime, site, operator, vendor e.t.c were created and endpoints were provided for client side to consume. Client can then download data in CSV format. This software makes it easy for the utility manager to persist record using mobile app or web app and be rest assured the headquater have access to it instantly or anytime needed. The project has service layer that performs some logics with the data before persisting. The program also send email notifications as necessary to all registered recipients of the operator. The project is developed using Java with Spring framework, lombok, Hibernate, Jackson, Kubernate etc.

RelayFX Desktop Application

This application back-end is written in Java and its front-end is written in FXML using the JavaFX technology. It is a control solution designed to control target device remotely via a bluetooth or zigbee serial transreceiver module connected to the operational desktop computer via USB. The target device shall have an embedded board as an intermediary between its transreceiver and the device control switch board. If your embedded board is an arduino, You can as well download the arduino code for this project here. You are free to download this code, study and edit to suit you. The operational PC must run JRE 1.8 or above.

Fingerprint Attendance App

This application is written in Java using the swing technology. It is developed to basically solve proxy attendance taking in any institution. Solar Mini Grid

Management of 85KWp solar mini-grid owned by Rubitec Nigeria Limited and located at Gbamugbamu village, Ijebu-Ode. I serve as the Utility Manager running the grid 24/7, ensuring utilization of generarted energy to create more revenue at minimal running cost. I coordinate the sales of power as well. My responsibility include solving problem as fast as possible to minimize down time. In the course of serving the company, I begin to develop custom mini-grid management software aimed at solving record persistance into remote database and making this record available for better decisions making. The software also automatically send email notifications. I also introduced a software solution that allow mass shutdown and restoration of customers' meters, a functionality not readily available on the platform used by the company.
